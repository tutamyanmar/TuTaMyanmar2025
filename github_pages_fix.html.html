GitHub Pages 404 Error Solution

သင့် GitHub Pages site ကို 404 error မတက်အောင် ဖြေရှင်းနည်းများ

တွေ့ရှိရသော Error

404 There isn't a GitHub Pages site here. If you're trying to publish one, read the full documentation to learn how to set up GitHub Pages for your repository, organization, or user account. GitHub Status — @githubstatus

ဒီ error ကို GitHub Pages site ကို မှန်ကန်စွာ မတည်ဆောက်ထားတဲ့အခါမှာ တွေ့ရတတ်ပါတယ်။

ဖြစ်နိုင်ခြေရှိသော အကြောင်းရင်းများ

1. Repository Name မှားယွင်းနေခြင်း

GitHub Pages အတွက် repository name ကို မှန်ကန်စွာ မပေးထားခြင်း။

2. GitHub Pages ကို Enable မလုပ်ရသေးခြင်း

Repository settings ထဲမှာ GitHub Pages feature ကို ဖွင့်မထားခြင်း။

3. Branch သို့ Folder မှားယွင်းနေခြင်း

မှန်ကန်တဲ့ branch (သို့) folder ကို ရွေးချယ်မထားခြင်း။

4. Build Process မအောင်မြင်ခြင်း

Site build process မှာ error ရှိနေခြင်း (Jekyll build errors)။

5. Custom Domain Configuration ပြဿနာ

Custom domain သုံးထားရင် DNS configuration မှားယွင်းနေခြင်း။

ဖြေရှင်းနည်းများ

Solution 1: Repository Name ကို မှန်ကန်အောင်ထားခြင်း

User/Organization Pages အတွက် repository name ကို username.github.io format နဲ့ပေးရပါမယ်။

1
Repository name ကို yourusername.github.io ပုံစံဖြစ်အောင် ပြောင်းပါ
2
သင့် GitHub username ကို အစားထိုးပါ (ဥပမာ - tutamyanmar.github.io)
3
Repository ကို Public အဖြစ်ထားပါ

Solution 2: GitHub Pages ကို Enable လုပ်ခြင်း

Repository settings ထဲမှာ GitHub Pages ကို enable လုပ်ဖို့လိုပါတယ်။

1
သင့် repository ထဲသို့သွားပါ
2
Settings tab ကိုနှိပ်ပါ
3
Sidebar မှ Pages ကိုရွေးပါ
4
Branch အောက်မှာ main (or master) ကိုရွေးပါ
5
Save ကိုနှိပ်ပါ

Solution 3: index.html ဖိုင်ထည့်ခြင်း

သင့် repository ထဲမှာ index.html ဖိုင်ရှိရပါမယ်။

<!DOCTYPE html> <html lang="my"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>My GitHub Page</title> </head> <body> <h1>မင်္ဂလာပါ! ကျွန်တော့် GitHub Page ကိုကြိုဆိုပါတယ်</h1> <p>ဒီ site ကို GitHub Pages နဲ့ တည်ဆောက်ထားတာဖြစ်ပါတယ်။</p> </body> </html>

စစ်ဆေးရန် အချက်များ

✓ Repository name မှန်ရဲ့လား (username.github.io)
✓ GitHub Pages enabled လုပ်ထားရဲ့လား
✓ index.html ဖိုင်ရှိရဲ့လား
✓ Repository က Public ဖြစ်ရဲ့လား
✓ အနည်းဆုံး 5-10 မိနစ်စောင့်ပြီးပြီလား (build process အတွက်)

သတိပြုရန်

GitHub Pages က site ကို build လုပ်ဖို့ အချိန်အနည်းငယ်ယူပါတယ်။ Setting ပြင်ပြီး ၁၀ မိနစ်ခန့်စောင့်ပါ။

အခြားဖြေရှင်းနည်းများ

1. GitHub Status ကိုစစ်ဆေးပါ

တစ်ခါတရံ GitHub ဘက်ကပြဿနာရှိနေတတ်ပါတယ်။ GitHub Status ကိုစစ်ဆေးပါ။

2. Custom Domain သုံးထားလျှင်

Custom domain သုံးထားရင် DNS settings မှန်ကန်အောင် ပြန်စစ်ပါ။ CNAME record ရှိရပါမယ်။

3. Jekyll Build Errors

Jekyll site ဆိုရင် _config.yml ဖိုင်မှာ error ရှိမရှိစစ်ပါ။

GitHub Pages ဆိုင်ရာ အကူအညီများ: GitHub Pages Documentation

အခြားပြဿနာများအတွက်: GitHub Support
